# visvoai-cli

A terminal-native coding agent built on `visvoai-core` + `visvoai-ai`: a Textual
TUI (interactive REPL) plus a single-shot mode. The agent reads/edits files, runs
shell commands, searches the web, and renders its work â€” streaming replies, tool
activity, diffs, thinking, and mermaid diagrams â€” in the terminal.

Two surfaces, one console script (`visvoai`):
- `visvoai`            â†’ launch the Textual TUI
- `visvoai "prompt"`   â†’ single-shot: stream one turn to stdout

# Key Files
- `main.py` â†’ the `visvoai` entry point: no prompt â†’ `_launch_tui` (VisvoApp); a prompt â†’ `_run_single_shot` (CLIRuntime graph â†’ `astream_events` â†’ stdout). Both build the model via `visvoai.ai.build_chat_model`. Single-shot is headless: `--yes` auto-approves mutations; without it the gated tool set DENIES mutations except those `[permissions]` pre-authorizes (path confinement applies either way).
- `app.py` â†’ `VisvoApp` â€” the Textual app: shell (CSS, bindings, compose/lifecycle, theme, welcome, quit/clear) + shared turn/HITL primitives (`_begin_turn`/`_mount_block`/`_tool_node`, `ask_choice`/`ask_form`/`ask_text`). Composed from mixins. Owns `_hitl_mode` + `action_cycle_hitl_mode` (shift+tab). Overrides `notify` to suppress info-level toasts (warning/error still show).
- `agent_turn.py` / `commands.py` / `sessions.py` / `render.py` / `demo.py` / `rewind.py` â†’ VisvoApp mixins (real turn, slash commands, resume/screens, answer+mermaid rendering, the test-only mock, git-structured history). Turn failures render via `ErrorBlock` (classified auth/network/model by `_classify_turn_error`), not a SystemNote. Persistence is **crash-durable**: the human message and each completed AI/Tool message are flushed via `_persist_turn` (idempotent tail-append) AS they stream, so a hard crash mid-turn loses nothing. The durable log is append-only and never trimmed; a dangling tool_call left by a crashed/errored turn is stripped at point-of-use by `_sanitize_thread` when building the model state (also covers resumed threads).
- `agent.py` â†’ pipeline glue: `build_agent_graph` (provider â†’ CLIRuntime graph), chunk classification, `SYSTEM_PROMPT`, deployment/usage/cost views. Textual-free.
- `runtime.py` â†’ `CLIRuntime(AgentRuntime)` â€” agentâ†’tools loop, no interrupts/checkpointer; when built with a `ContextAssembler` it overrides `_build_agent_node` for per-turn prompt assembly (still honouring the soft-step-cap finalize), else falls back to the core static-prompt node.
- `context.py` â†’ `CLIContext(RuntimeContext)` â€” cwd / terminal width; no auth/datastore.
- `store.py` â†’ folder-per-conversation persistence (no DB). Conversation: `meta.json` (title, `active_branch`, msg_count) + `checkpoints.jsonl` (the shared, append-only, immutable REGISTRY: checkpoint `id â†’ shadow commit`). Branches: `branches/<name>/` each with `thread.jsonl` + `receipts.jsonl` + `timeline.jsonl` (ordered `{checkpoint_id, message_index, kind, label}` â€” that branch's turn/toolâ†’checkpoint view) + `meta.json` (`tip`, `forked_from`). Conversation-level convenience (`save_conversation`/`append_messages`/`append_receipt`/`read_receipts`) delegates to the active branch. `project_root(cwd)` is the stable shadow-repo work-tree anchor. `find_project_id(cwd)` (read-only) + `has_conversations(pid)` (read-only) let the welcome banner detect launch state without writing the project anchor.
- `state.py` â†’ UI state, NOT in-repo. Per-project at `~/.visvoai/projects/<pid>/state.json` (`last_seen_version` for the "what's new" panel) + per-USER global at `~/.visvoai/state.json` (`used` features + `shown` one-time flags) driving adaptive tips & coachmarks. `record_used(feature)` / `used_features()`; `mark_shown(key)` (True the first time) / `was_shown(key)`. All reads are side-effect-free; writes are atomic + best-effort.
- `cli_changelog.py` â†’ parses the bundled `assets/CHANGELOG.md` via regex; exposes `parse_changelog()`, `new_since(last_seen)`, `one_line_summary(entry)`, `current_version()` (from installed metadata). Version compare is tuple-based so `'0.10.0' > '0.4.0'` is correct (string compare is not).
- `checkpoints.py` â†’ `ShadowRepo`: a separate GIT_DIR (`~/.visvoai/projects/<pid>/checkpoints.git`) with the project root as work tree â€” snapshots the whole tree per tool-batch/turn (plumbing: add -A â†’ write-tree â†’ commit-tree, explicit parent, content-addressed dedup), TRUE-restores it, and supports refs / worktree fork / bundle. Never touches the user's own `.git`; honours `.gitignore` + default excludes (incl. `.visvoai` secrets). `available()` False â†’ callers skip checkpointing.
- `rewind.py` â†’ `RewindMixin`: records checkpoints during a turn (best-effort, never breaks a turn); `/rewind` (Ctrl+B) restore-or-branch; `/branch` switch/fork timelines; `/fork` worktree + seeded conversation; `/export` transcript or bundle; `/log` the chain. A branch reconstructs itself ONLY from its own folder (its `timeline.jsonl`), resolving a checkpoint id against the shared immutable registry â€” never reading another branch's mutable state. Branch-from = deep-copy a branch folder + truncate (both timelines kept, fully isolated); `forked_from` is provenance only. See `cli-git-structure`.
- `keys.py` â†’ layered API-key resolution + storage: `load_keys_into_env(cwd)` (env > project secrets > global config), `set_key(provider, key, scope, cwd)` (0600, auto-gitignore for project).
- `catalog.py` â†’ `install_cli_catalog()` â€” at startup installs `build_catalog([BakedSource(), RemoteModelsDevSource(cache)])` as the default registry so the picker shows the live models.dev catalog (cached `~/.visvoai/cache/models.json`, offline-tolerant). `--refresh-models` re-fetches. The picker (`agent.chat_deployments`) shows curated baked deployments always + a provider's full catalog only when keyed.
- `gated_tools.py` â†’ edit/write/shell behind a permission gate; edit/write also path-confined (`pathguard.confine`, boundary checked BEFORE the prompt); web tools ungated.
- `pathguard.py` â†’ `resolve_roots(cwd)` (cwd + `.visvoai/config.toml [permissions].write_roots`) + `confine(path, roots)` â†’ realpath-resolved, escape/symlink/`.git`-blocked; raises `PathDenied`. Confines WRITES only (reads free); `run_shell` is gate-governed, not confinable.
- `permissions.py` â†’ `PermissionPolicy.auto_allow(tool, args)` from `.visvoai/config.toml [permissions]` (`allow_shell` command prefixes, `allow_write` path globs); `load_policy(cwd)` layers globalâ†’project. Pre-authorized ops skip the gate.
- `hitl_modes.py` â†’ `HITLMode` (NORMAL/AUTO_EDIT/ACCEPT_ALL): `next()` cycles, `auto_approves(tool)` (auto-edit = file writes only, shell stays gated), `chip` (None in NORMAL). Session-only; cycled by shift+tab (`action_cycle_hitl_mode`, priority binding) or `/mode`. Relaxes approval only â€” path confinement is independent. Concurrent tool approvals are serialized by `app._hitl_lock` (ToolNode runs tool_calls in parallel) with a re-check inside the lock so a sibling's "allow all session" skips the redundant prompt.
- `gitio.py` â†’ real git: working-tree status / `status_summary` (diff-free per-turn snapshot) / stage / unstage / commit / project files.
- `context/` â†’ configurable per-turn system-prompt assembly (`build_assembler`); slots into `CLIRuntime._build_agent_node` (see `context/`).
- `mermaid.py` â†’ ```mermaid fence â†’ segments + a self-contained HTML viewer (pure helpers).
- `theme.py` / `grid.py` / `termbg.py` â†’ 12-palette theme (reads `palette_tokens.json`), grid alignment, terminal-bg detection.
- `tools/` â†’ the agent tool set (files/shell/web split; see `tools/`).
- `widgets/`, `screens/` â†’ reusable Textual widgets and full-screen views (each has its own AGENTS.md).
- `palette_tokens.json`, `assets/` â†’ shipped package data (see `[tool.setuptools.package-data]`).

# Key Classes / Functions
- `VisvoApp(App)` â†’ the Textual app; exposed lazily via `visvoai.cli.__getattr__` so `import visvoai.cli` / single-shot don't pull in Textual.
- `CLIRuntime(AgentRuntime)` â†’ no `_extend_graph` additions; optional `_build_agent_node` override for the context-assembly pipeline (`__init__(assembler=â€¦)`).
- `cli` (main.py) â†’ Click command, the `visvoai` console script; routes promptâ†’single-shot, no-promptâ†’TUI.
- `build_agent_graph()` (agent.py) â†’ builds the graph the TUI streams; `build_chat_model` applies the deployment's default thinking.

# Conventions
- Pure public-package consumer: imports only `visvoai-ai`, `visvoai-core`, relative modules, and declared third-party deps (textual, rich, click, pygments, python-dotenv, langchain-google-genai). NEVER a private/platform module.
- No private names in shipped text (docstrings/comments/AGENTS.md) â€” boundary rule 2.
- Every widget owns its styling via Textual `DEFAULT_CSS`; colours come only from `theme.py`.
- `app.py` holds the shell + shared primitives; feature concerns live in mixins.
- No graph interrupts/checkpointer â€” the CLI is synchronous (the human is in the loop).

# Gotchas
- API keys: `main._bootstrap_env(cwd)` (both surfaces) loads `.env` then fills `os.environ` from the stored layers. Precedence is env > project (`.visvoai/secrets.toml`, gitignored) > global (`~/.visvoai/config.toml` `[api_keys]`); an exported var always wins. Keys NEVER go in the committed `.visvoai/config.toml`. Per-provider: the active model's provider determines which `{PROVIDER}_API_KEY` is needed; web search/fetch always need Gemini's. Add a key via `/login` (TUI) or `visvoai --set-key <provider>`.
- `astream_events` is called with an explicit `recursion_limit=100` (both surfaces) â€” the LangGraph default 25 crashes a deep turn before the core graph's soft step cap can force a clean finalize.
- The mock showcase (DemoMixin + mock.py) is kept for tests but is NOT reachable from the menu, by typing `/demo`, or any key binding â€” tests drive it via `run_command`.
- `theme.py` reads `palette_tokens.json` from its own directory at import time â†’ it must ship as package data (it does).
- Tests live in `tests/` and run in the package's own env (`uv run --extra dev pytest`); `[tool.uv.sources]` resolves the sibling packages by path for dev (pip/PyPI ignore that table).
- **Self-driving guidance** teaches by context and never nags: spinner tips are adaptive (`widgets.conversation.adaptive_tips(used)` drops a feature's tip once `state.record_used` has logged it); post-turn nudges + coachmarks self-silence (`_nudge_text` is pure/tested; coachmarks gated by `state.mark_shown`); `/tour` + `/help` + launch-state welcome + "what's new" are the discovery surfaces; unknown `/cmd` â†’ `commands._closest_command` (alias table then fuzzy). The proactive launch scan does real-git I/O so it's gated on `self.is_headless` (never runs under test). Keep the restraint: info-severity toasts are suppressed, panels self-silence â€” that's what stops it feeling naggy.
- Git-structured history (rewind/branch/fork/export) is **best-effort**: a `CheckpointError` (or missing git) is always swallowed so it never breaks a turn â€” `ShadowRepo.available()` gates it, and checkpoint tests `skipif` git is absent. TRUE restore is destructive on the active branch, but every checkpoint commit keeps a per-checkpoint ref (`refs/visvoai/<cid>/cp/<id>`) so discarded code stays recoverable; branching keeps both timelines outright and is **isolated** (deep-copied â€” later changes to the parent never touch a fork). The shadow repo is per-project (shared, deduped, content-addressed commits); the checkpoint registry is shared and **immutable** (an id always maps to the same commit), which is what makes cross-branch id resolution safe. `/clear` resets the checkpoint tip so a fresh conversation re-baselines.
